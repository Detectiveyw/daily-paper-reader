## 摘要
密码学安全神经网络推理通常依赖于安全多方计算 (MPC) 等安全计算技术，使云服务器能够在不解密的情况下处理客户端输入。尽管先前的隐私保护推理系统将网络优化与 MPC 进行了协同设计，但它们仍然缓慢且昂贵，限制了实际部署。一个主要的瓶颈是它们对所有加密输入都使用单一、固定的 Transformer 模型，忽略了不同的输入需要不同的模型大小来平衡效率和准确性。我们提出了 SecureRouter，这是一个端到端的加密路由和推理框架，通过在加密状态下进行输入自适应的模型选择，加速了安全 Transformer 推理。SecureRouter 建立了一个统一的加密流水线，将安全路由器与 MPC 优化的模型池集成，在保持完整的数据和模型机密性的同时，实现了协调的路由、推理和协议执行。该框架包括训练阶段和推理阶段的组件：一个感知 MPC 成本的安全路由器，用于根据加密特征预测每个模型的效用和成本；以及一个 MPC 优化的模型池，其架构和量化方案经过协同训练，以最小化 MPC 通信和计算开销。与先前的工作相比，SecureRouter 在准确率损失微乎其微的情况下实现了 1.95 倍的延迟降低，为可扩展且高效的安全 AI 推理提供了一条切实可行的路径。我们的开源实现可在以下网址获得：https://github.com/UCF-ML-Research/SecureRouter。

## Abstract
Cryptographically secure neural network inference typically relies on secure computing techniques such as Secure Multi-Party Computation (MPC), enabling cloud servers to process client inputs without decrypting them. Although prior privacy-preserving inference systems co-design network optimizations with MPC, they remain slow and costly, limiting real-world deployment. A major bottleneck is their use of a single, fixed transformer model for all encrypted inputs, ignoring that different inputs require different model sizes to balance efficiency and accuracy. We present SecureRouter, an end-to-end encrypted routing and inference framework that accelerates secure transformer inference through input-adaptive model selection under encryption. SecureRouter establishes a unified encrypted pipeline that integrates a secure router with an MPC-optimized model pool, enabling coordinated routing, inference, and protocol execution while preserving full data and model confidentiality. The framework includes training-phase and inference-phase components: an MPC-cost-aware secure router that predicts per-model utility and cost from encrypted features, and an MPC-optimized model pool whose architectures and quantization schemes are co-trained to minimize MPC communication and computation overhead. Compared to prior work, SecureRouter achieves a latency reduction by 1.95x with negligible accuracy loss, offering a practical path toward scalable and efficient secure AI inference. Our open-source implementation is available at: https://github.com/UCF-ML-Research/SecureRouter

#### 1. 核心问题与研究动机
*   **核心问题**：基于安全多方计算（MPC）的隐私保护神经网络推理虽然能保护数据隐私，但计算和通信开销巨大。现有的系统（如 MPCFormer, SecFormer）通常对所有输入使用单一、固定的模型，忽略了不同输入（简单 vs. 困难）对计算资源需求的差异，导致了严重的效率瓶颈。
*   **研究动机**：在明文推理中，输入自适应路由（Model Routing）已成熟，但在加密环境下实现路由面临三大挑战：
    1.  **不可见性**：路由决策必须在加密数据上进行，且不能泄露路由选择。
    2.  **成本度量差异**：MPC 的成本由通信轮数和非线性操作主导，而非传统的 FLOPs。
    3.  **端到端集成**：需要将加密路由、不经意模型检索和安全推理无缝集成。

#### 2. 方法论
*   **核心思想**：提出 SecureRouter 框架，通过一个“感知 MPC 成本的安全路由器”动态地从“MPC 优化的模型池”中为每个加密输入选择最合适的模型。
*   **关键技术细节**：
    *   **双阶段设计**：
        *   **离线阶段**：协同训练路由器和模型池。路由器使用 Gumbel-Softmax 技巧实现离散选择的可微化。
        *   **在线阶段**：利用加法秘密共享（Additive Secret Sharing）处理输入，通过安全 Argmax 确定模型索引，并利用不经意传输（OT）检索模型参数。
    *   **复合损失函数**：
        *   $L_{task}$：确保分类准确率。
        *   $L_{balance}$：防止所有输入都路由到同一个模型（模式崩溃）。
        *   $L_{cost}$：根据 MPC 环境下的实际推理时间（而非参数量）对高开销模型进行惩罚。
    *   **MPC 优化模型池**：包含不同规模的 BERT 变体（Tiny, Base, Large），其架构和量化方案针对 MPC 协议进行了协同优化。

*   **自适应性**：路由器能成功识别“简单”样本并分配给小模型，而在复杂语义任务上则倾向于调用大模型。

#### 7. 优点（亮点）
*   **首创性**：这是首个将输入自适应路由引入 MPC 环境的端到端 Transformer 推理框架。
*   **MPC 成本感知**：不同于以往只看参数量的优化，该方法直接将 MPC 协议的实际延迟建模进损失函数。
*   **高效集成**：通过 OT 和秘密共享技术，实现了在不泄露“选了哪个模型”的前提下完成推理。

#### 8. 不足与局限
*   **模型多样性有限**：实验主要集中在 BERT 变体上，对于更现代的超大规模生成式模型（如 Llama 系列）的适用性尚未验证。
*   **安全假设**：目前基于半诚实（Semi-honest）威胁模型，在恶意（Malicious）环境下的安全性及性能表现有待研究。
*   **离线开销**：虽然在线推理变快了，但离线阶段需要协同训练多个专家模型和路由器，初期计算成本较高。

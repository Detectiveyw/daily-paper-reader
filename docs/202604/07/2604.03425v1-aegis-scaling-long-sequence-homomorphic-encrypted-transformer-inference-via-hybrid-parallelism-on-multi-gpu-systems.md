## 摘要
全同态加密 (FHE) 实现了隐私保护的 Transformer 推理，但长序列加密 Transformer 会迅速超过单 GPU 的显存容量，因为编码后的权重本身很大，且加密激活值随序列长度增长迅速。因此，多 GPU 执行变得不可避免，但由于通信是由应用层聚合和加密层 RNS 耦合共同引起的，扩展性仍然面临挑战。现有方法要么在设备间频繁同步，要么在设备间复制加密张量，导致过多的通信和延迟。我们提出了 AEGIS，一种用于多 GPU 平台上可扩展长序列加密 Transformer 推理的应用-加密引导推理系统。AEGIS 根据 Transformer 数据流和 CKKS 多项式耦合共同产生的密文依赖关系来推导设备放置策略，将模数相干（modulus-coherent）和令牌相干（token-coherent）的数据进行同地协作，从而仅在应用依赖需要时才引入通信，同时通过重新排列多项式算子使剩余的集合通信与计算重叠。在 2048 个令牌的输入下，与现有先进设计相比，AEGIS 在前馈网络中减少了高达 57.9% 的 GPU 间通信，在自注意力机制中减少了 81.3%。在四块 GPU 上，它实现了高达 96.62% 的扩展效率、3.86 倍的端到端加速以及 69.1% 的单设备显存减少。这些结果表明，协同的应用-加密并行是实现可扩展同态 Transformer 推理的实用基础。

## Abstract
Fully Homomorphic Encryption (FHE) enables privacy-preserving Transformer inference, but long-sequence encrypted Transformers quickly exceed single-GPU memory capacity because encoded weights are already large and encrypted activations grow rapidly with sequence length. Multi-GPU execution therefore becomes unavoidable, yet scaling remains challenging because communication is jointly induced by application-level aggregation and encryption-level RNS coupling. Existing approaches either synchronize between devices frequently or replicate encrypted tensors across devices, leading to excessive communication and latency.   We present AEGIS, an Application-Encryption Guided Inference System for scalable long-sequence encrypted Transformer inference on multi-GPU platforms. AEGIS derives device placement from ciphertext dependencies jointly induced by Transformer dataflow and CKKS polynomial coupling, co-locating modulus-coherent and token-coherent data so that communication is introduced only when application dependencies require it, while reordering polynomial operators to overlap the remaining collectives with computation.   On 2048-token inputs, AEGIS reduces inter-GPU communication by up to 57.9% in feed-forward networks and 81.3% in self-attention versus prior state-of-the-art designs. On four GPUs, it achieves up to 96.62% scaling efficiency, 3.86x end-to-end speedup, and 69.1% per-device memory reduction. These results establish coordinated application-encryption parallelism as a practical foundation for scalable homomorphic Transformer inference.

### 论文总结：AEGIS — 多 GPU 系统上的长序列同态加密 Transformer 推理扩展方案

#### 1. 核心问题与整体含义（研究动机和背景）
全同态加密（FHE）允许在不解密的情况下对数据进行计算，是实现隐私保护机器学习（MLaaS）的关键技术。然而，将 FHE 应用于 Transformer 模型（如 BERT、LLM）面临严重的**显存爆炸**和**通信瓶颈**：
*   **显存压力**：加密后的模型权重和激活值比明文大几个数量级。例如，2048 个令牌的 BERT 推理需要超过 700GB 的编码权重和 110GB 的密文激活值，远超单张 GPU 的容量。
*   **通信挑战**：现有的多 GPU 并行方案（如肢体并行 Limb Parallelism 或简单的张量并行）在处理加密数据时效率极低。加密算法特有的 RNS 耦合和应用层的令牌聚合共同导致了海量的跨设备数据交换，甚至会出现增加 GPU 反而变慢的现象。
**AEGIS 的目标**是设计一个应用与加密协同引导的推理系统，通过优化数据放置和算子调度，实现长序列加密 Transformer 在多 GPU 上的高效扩展。

#### 2. 方法论：核心思想与关键技术
AEGIS 的核心思想是**联合编排应用层数据流与底层加密原语的执行**，主要包含以下技术细节：
*   **依赖感知的工作负载划分**：
    *   **模数相干放置（Modulus-coherent）**：针对 CKKS 方案中的 RNS 耦合，将参与同一密钥交换或旋转链的多项式组放置在同一设备上，消除指令级的跨设备同步。
    *   **令牌相干放置（Token-coherent）**：识别应用层的令牌边界，将同一令牌的不同切片尽可能共置，减少嵌入维度缩减（Reduction）时的通信。
*   **前馈网络（FFN）与自注意力机制的并行化**：
    *   在 FFN 中，强制要求属于同一输出组的权重切片共置，避免产生部分槽位利用率低导致的冗余通信。
    *   在自注意力中，采用“部分计算-分布式 Softmax-输出聚合”的三阶段策略，仅在必要时插入集合通信。
*   **多项式算子重排（Operator Reordering）**：
    *   通过重排旋转偏移量的计算顺序，使 `ReduceScatter` 或 `AllGather` 等集合通信操作能够与计算任务重叠，从而掩盖通信延迟。
*   **编译优化 pass**：在中间层分析令牌和 RNS 依赖，自动插入通信算子并生成确定性的多 GPU 执行计划。

#### 3. 实验设计
*   **模型与任务**：使用 **BERT-Base** 模型在 **SST-2** 数据集上进行端到端推理验证。
*   **序列长度**：测试了从 64 到 **2048** 个令牌的不同长度。
*   **对比基准（Benchmarks）**：

    2.  **Cinnamon**：针对 ASIC/FPGA 设计的混合并行策略（移植到 GPU）。
    3.  **Hydra**：采用列张量并行的扩展方案。

#### 5. 实验数量与充分性
*   **实验维度**：涵盖了端到端延迟、单层组件效率、通信量对比、显存占用分析以及弱扩展性分析。
*   **充分性**：实验对比了多种 SOTA 并行策略，并针对长序列（2048 tokens）这一极端场景进行了压力测试。通过消融实验区分了“数据放置优化”和“算子重排优化”的各自贡献，实验设计较为客观、公平。

#### 6. 主要结论与发现
*   **通信大幅减少**：在 2048 令牌下，AEGIS 相比现有方案在 FFN 中减少了 57.9% 的通信，在自注意力中减少了 81.3%。
*   **高扩展效率**：在 4 卡 A100 上实现了 **96.62%** 的扩展效率，端到端加速比达到 **3.86 倍**。
*   **显存优化显著**：相比于需要复制激活值的方案，AEGIS 将单设备显存占用降低了高达 **69.1%**，使得在有限显存的 GPU 上运行超长序列成为可能。
*   **打破肢体并行限制**：传统的肢体并行受限于模数链长度（通常只能扩展到 14-16 卡），而 AEGIS 的扩展性随序列长度增长，具有更好的弱扩展性。

#### 7. 优点
*   **跨层协同设计**：首次深入探讨了 Transformer 应用语义与 CKKS 加密原语之间的耦合关系，并给出了系统化的放置规则。
*   **确定性调度**：通过静态编译分析实现通信掩盖，无需运行时的动态启发式搜索。
*   **显存高效**：通过分片而非复制密文，解决了 FHE 推理中最核心的显存溢出（OOM）问题。

#### 8. 不足与局限
*   **模型覆盖范围**：虽然理论上适用于所有 Transformer，但实验主要集中在 BERT-Base（Encoder 架构），对于超大规模的 Decoder-only 模型（如 GPT-3 级别）的测试尚不充分。
*   **绝对延迟依然较高**：尽管实现了多卡加速，但受限于 FHE 本身的计算复杂度，2048 令牌的推理时间仍以千秒计，距离实时交互仍有距离。
*   **硬件依赖**：部分优化效果依赖于高速互联（如 NVLink），在 PCIe 带宽受限的环境下，通信掩盖的效果可能会打折扣。

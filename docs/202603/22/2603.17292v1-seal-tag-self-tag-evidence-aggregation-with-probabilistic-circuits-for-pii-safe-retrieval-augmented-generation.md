## 摘要
检索增强生成 (RAG) 系统引入了一个关键漏洞：上下文泄露，即攻击者利用指令遵循能力，通过自适应提取手段窃取个人身份信息 (PII)。目前的防御措施在语义效用和延迟之间被迫进行僵化的权衡。我们提出了 SEAL-Tag，这是一种保护隐私的运行时环境，通过“先验证后路由”范式解决了这一问题。SEAL-Tag 引入了 SEAL-Probe 协议，将审计转化为结构化的工具使用操作，使模型在生成草稿的同时生成一个可验证的 PII 证据表 (PET)。为了裁定这些证据，我们采用了概率电路 (PC)，通过强制执行可验证的逻辑约束来实现稳健的决策。为了克服隐私“冷启动”问题，我们引入了 S0--S6 锚定合成流水线，生成具有高保真度和来源可溯的 RAG 交互数据。我们将其与两阶段课程学习相结合，首先优化实体检测，然后使模型对齐严格的审计协议。我们的评估表明，SEAL-Tag 建立了新的帕累托前沿，在保持与不安全基准模型相当的效用和速度的同时，将自适应泄露降低了 8 倍以上。

## Abstract
Retrieval-Augmented Generation (RAG) systems introduce a critical vulnerability: contextual leakage, where adversaries exploit instruction-following to exfiltrate Personally Identifiable Information (PII) via adaptive extraction. Current defenses force a rigid trade-off between semantic utility and latency. We present SEAL-Tag, a privacy-preserving runtime environment that resolves this via a Verify-then-Route paradigm. SEAL-Tag introduces the SEAL-Probe protocol, transforming auditing into a structured tool-use operation where the model generates a verifiable PII-Evidence Table (PET) alongside its draft. To adjudicate this evidence, we employ a Probabilistic Circuit (PC) that enforces verifiable logical constraints for robust decision-making. To overcome the privacy "Cold Start" problem, we introduce the S0--S6 Anchored Synthesis Pipeline, generating high-fidelity, provenanced RAG interactions. We pair this with a Two-Stage Curriculum that first optimizes for entity detection before aligning the model to the rigorous audit protocol. Our evaluation demonstrates that SEAL-Tag establishes a new Pareto frontier, reducing adaptive leakage by over 8$\times$ while matching the utility and speed of unsafe baselines.

### 2. 方法论
SEAL-Tag 提出了 **“先验证后路由”（Verify-then-Route）** 范式，核心思想是将隐私审计从模糊的生成任务转变为结构化的工具调用操作。
*   **SEAL-Probe 协议**：强制执行三阶段生成契约：`<ANSWER>`（草稿生成）→ `<PET>`（PII 证据表生成）→ `<FINAL>`（最终路由输出）。
*   **PII 证据表 (PET)**：模型生成的结构化 JSON，包含四个维度：
    1.  **实体来源与暴露**：标记实体类型、可见范围（答案/查询/上下文）及来源文档索引。
    2.  **链接性与马赛克风险**：评估多个良性属性组合后是否会导致身份重识别。
    3.  **共识与自洽性**：检查草稿答案与检索证据是否逻辑一致。
    4.  **意图与攻击遥测**：识别用户是否存在注入或欺骗意图。
*   **概率电路 (PC) 决策头**：取代不透明的神经网络分类器。PC 是一种可处理的生成模型，能够强制执行**单调性逻辑约束**（即风险证据增加时，风险评分绝不会降低），确保决策的数学可解释性和校准性。
*   **S0–S6 锚定合成流水线**：解决隐私数据“冷启动”问题。通过从 PII 模式库抽取锚点、模拟世界背景、注入攻击片段等 7 个步骤，生成高质量、带标签的合成 RAG 交互数据。
*   **两阶段课程学习**：
    *   **阶段 I（感知）**：优化模型对 PII 实体的敏感度（高召回率）。
    *   **阶段 II（对齐）**：训练模型遵循严格的 PET 协议格式。

### 6. 主要结论与发现

*   **消除“安全税”**：相比于 PrivacyMind 等导致效用大幅下降的方法，SEAL-Tag 几乎不损失 QA 性能。
*   **高可靠性**：PC 决策头的预期校准误差（ECE）仅为 0.03，远优于神经分类器（0.16），有效解决了“过度自信的泄露”问题。
*   **极低延迟**：相比于 LLM Judge 增加的 1450ms 延迟，SEAL-Tag 仅增加 **18ms**，完全满足实时边缘部署需求。

### 7. 优点
*   **架构创新**：将隐私审计转化为结构化工具调用，解决了 LLM 内部状态不可见的问题。
*   **逻辑严密**：引入概率电路保证了决策的单调性和可验证性，这是传统深度学习模型难以实现的。
*   **数据合成方案**：S0-S6 流水线为隐私研究提供了一种无需真实敏感数据即可训练高性能审计器的方法。
*   **实用性强**：极低的延迟和对小参数模型（3B）的良好支持，使其具备极高的工业落地价值。

### 8. 不足与局限
*   **格式依赖**：虽然采用了两阶段训练，但较小规模的模型在面对极长上下文时，仍可能存在 JSON 格式崩溃或实体遗漏的风险。
*   **合成偏差**：尽管合成数据质量高，但真实世界中的隐私定义（如特定行业的合规要求）可能比合成模式更复杂。
*   **多模态限制**：目前研究集中在文本 RAG，尚未探讨图像或多模态检索中的 PII 泄露问题。
*   **应用范围**：主要针对已知类型的 PII，对于某些高度隐蔽、非结构化的商业机密保护能力尚待进一步验证。

## 摘要
端侧大语言模型（LLM）经历了爆发式增长，与云端 LLM 相比，它们提供了更高的隐私性和可用性。在 LLM 推理过程中，模型权重和用户数据都非常宝贵，攻击者甚至可能通过攻破操作系统内核来窃取这些数据。ARM TrustZone 是移动设备上事实上的硬件隔离技术，用于保护敏感应用免受受损操作系统的侵害。然而，由于 TrustZone 对内存和 NPU 的隔离缺乏灵活性，使用 TrustZone 保护 LLM 推理会产生显著的开销。为了应对这些挑战，本文提出了 FlexServe，这是一种用于移动设备的快速且安全的 LLM 推理系统。它首先引入了一种灵活资源隔离机制，用以构建灵活安全内存（Flex-Mem）和灵活安全 NPU（Flex-NPU）。内存页和 NPU 均可在非保护模式和保护模式之间高效切换。基于这些机制，FlexServe 在 TrustZone 的安全世界（secure world）内设计了一个快速且安全的 LLM 推理框架。文中引入了 LLM 感知内存管理和安全推理流水线以加速推理，并提出了一种多模型调度器来优化多模型工作流。我们实现了 FlexServe 的原型，并将其与两种基于 TrustZone 的基准（strawman）设计进行了比较。结果表明，与基准设计相比，FlexServe 在首字延迟（TTFT）方面实现了平均 10.05 倍的加速；与启用了流水线和安全 NPU 的优化基准设计相比，实现了平均 2.44 倍的 TTFT 加速。对于多模型智能体（agent）工作流，其端到端加速比相较于基准设计和优化基准设计分别达到了 24.30 倍和 4.05 倍。

## Abstract
Device-side Large Language Models (LLMs) have witnessed explosive growth, offering higher privacy and availability compared to cloud-side LLMs. During LLM inference, both model weights and user data are valuable, and attackers may even compromise the OS kernel to steal them. ARM TrustZone is the de facto hardware-based isolation technology on mobile devices, used to protect sensitive applications from a compromised OS. However, protecting LLM inference with TrustZone incurs significant overhead due to its inflexible isolation of memory and the NPU. To address these challenges, this paper introduces FlexServe, a fast and secure LLM serving system for mobile devices. It first introduces a Flexible Resource Isolation mechanism to construct Flexible Secure Memory (Flex-Mem) and Flexible Secure NPU (Flex-NPU). Both memory pages and the NPU can be efficiently switched between unprotected and protected modes. Based on these mechanisms, FlexServe designs a fast and secure LLM inference framework within TrustZone's secure world. The LLM-Aware Memory Management and Secure Inference Pipeline are introduced to accelerate inference. A Multi-Model Scheduler is proposed to optimize multi-model workflows. We implement a prototype of FlexServe and compare it with two TrustZone-based strawman designs. The results show that FlexServe achieves an average $10.05\times$ speedup in Time to First Token (TTFT) compared to the strawman, and an average $2.44\times$ TTFT speedup compared to an optimized strawman with pipeline and secure NPU enabled. For multi-model agent workflows, the end-to-end speedup is up to $24.30\times$ and $4.05\times$ compared to the strawman and optimized strawman, respectively.

*   **研究背景**：端侧大语言模型（LLM）在隐私保护和可用性方面具有优势，但其模型权重（高价值资产）和用户数据（敏感隐私）面临被受损操作系统（OS）内核窃取的风险。
*   **核心痛点**：现有的硬件隔离技术（如 ARM TrustZone）在保护 LLM 推理时存在严重性能瓶颈：
    1.  **内存隔离僵化**：TrustZone 要求安全内存必须是连续的物理内存，在内存受限的移动端分配 GB 级别的连续内存极其缓慢（如 8GB 需 6.44 秒）。
    2.  **NPU 无法共享**：NPU 通常被静态分配给普通世界，安全世界无法调用 NPU 加速，导致只能使用 CPU 推理，速度极慢。
    3.  **多模型挑战**：移动端 Agent 往往需要频繁切换多个模型，传统机制难以应对动态的内存需求。
*   **研究目标**：设计一个既能防御内核级攻击，又能实现接近原生性能的端侧 LLM 推理系统。

### 2. 方法论
FlexServe 结合了硬件虚拟化扩展与软件优化，核心组件包括：
*   **灵活资源隔离机制**：
    *   **Flex-Mem（灵活安全内存）**：利用 ARM 虚拟化扩展（EL2 模式）的二阶段页表（S2PT），实现页粒度（非连续）的内存保护。支持动态将普通内存页切换为安全页，并能根据内存压力进行回收。
    *   **Flex-NPU（灵活安全 NPU）**：通过构建 NPU 驱动沙箱，实现 NPU 在普通世界和安全世界之间的高效时间片切换，使安全推理能利用硬件加速。
    *   **按需保护（On-demand Protection）**：在没有安全任务时，通过 EL3 监控器关闭虚拟化功能，消除对普通应用的运行时开销。
*   **FlexServe 推理框架**：
    *   **安全推理流水线**：将内存分配、模型加载、解密和 NPU 计算重叠执行，隐藏 I/O 和预处理延迟。
    *   **LLM 感知内存管理**：采用层级权重缓存（优先保留前几层以消除启动停顿）和类似 PagedAttention 的 KV Cache 管理。
    *   **多模型调度器**：针对 Agent 工作流预测下一个可能调用的模型，并在后台预取权重。

### 6. 主要结论

### 7. 优点
*   **架构创新**：巧妙地结合了 TrustZone 的安全性和虚拟化扩展的灵活性，解决了 TEE 内存管理的长期痛点。
*   **软硬协同**：不仅在底层做隔离，还在应用层针对 LLM 的层级结构和 KV Cache 特性做了深度优化。
*   **实用性强**：在商用 SoC（RK3588）上实现了原型，证明了方案在现有移动硬件上的可行性。

### 8. 不足与局限
*   **硬件依赖**：方案高度依赖 ARM 虚拟化扩展（EL2），在不支持该特性的旧款或低端芯片上无法运行。

*   **GPU 缺失**：目前仅支持 CPU 和 NPU，尚未整合移动端同样常见的 GPU 算力。

## 摘要
检索增强生成 (RAG) 使大语言模型能够利用外部知识，但外包 RAG 服务会引发数据所有者和用户的隐私担忧。隐私保护 RAG 系统通过执行安全 Top-$k$ 检索来解决这些问题，这通常涉及通过安全排序来识别相关文档。然而，现有系统在支持任意 $k$ 值方面面临挑战，原因包括无法更改 $k$、新的安全问题或在大 $k$ 值下的效率下降。这是一个重大限制，因为现代长上下文模型通常在检索集较大时获得更高的准确率。我们提出了 $p^2$RAG，一种支持任意 Top-$k$ 检索的隐私保护 RAG 服务。与现有系统不同，$p^2$RAG 避免了对候选文档进行排序。相反，它使用交互式二分法来确定 Top-$k$ 文档集。在安全性方面，$p^2$RAG 在两个半诚实且互不串通的服务器上使用秘密共享，以保护数据所有者的数据库和用户的提示词。它通过强制执行限制和验证来防御恶意用户，并严格限制数据库的信息泄露。实验表明，在 $k = 16$ 到 $1024$ 的情况下，$p^2$RAG 比最先进的 PRAG 快 3 到 300 倍。

## Abstract
Retrieval-Augmented Generation (RAG) enables large language models to use external knowledge, but outsourcing the RAG service raises privacy concerns for both data owners and users. Privacy-preserving RAG systems address these concerns by performing secure top-$k$ retrieval, which typically is secure sorting to identify relevant documents. However, existing systems face challenges supporting arbitrary $k$ due to their inability to change $k$, new security issues, or efficiency degradation with large $k$. This is a significant limitation because modern long-context models generally achieve higher accuracy with larger retrieval sets. We propose $p^2$RAG, a privacy-preserving RAG service that supports arbitrary top-$k$ retrieval. Unlike existing systems, $p^2$RAG avoids sorting candidate documents. Instead, it uses an interactive bisection method to determine the set of top-$k$ documents. For security, $p^2$RAG uses secret sharing on two semi-honest non-colluding servers to protect the data owner's database and the user's prompt. It enforces restrictions and verification to defend against malicious users and tightly bound the information leakage of the database. The experiments show that $p^2$RAG is 3--300$\times$ faster than the state-of-the-art PRAG for $k = 16$--$1024$.

*   **核心问题**：如何在保护数据所有者（数据库）和用户（查询词）隐私的前提下，实现高效且灵活的检索增强生成（RAG）服务？
*   **研究动机**：
    *   **现有方案局限**：现有的隐私保护 RAG 系统通常将 Top-$k$ 检索视为安全排序问题，在大 $k$ 值（现代长上下文模型所需）下效率极低，且难以动态调整 $k$ 值。
    *   **长上下文需求**：研究表明，较大的检索集（大 $k$）能显著提升长文本 LLM 的准确率，但现有加密检索技术难以兼顾大 $k$ 值的性能与灵活性。

### 2. 方法论
*   **核心思想**：放弃高开销的“安全排序”，转而采用**交互式二分法（Interactive Bisection）**来确定距离阈值，从而识别 Top-$k$ 文档集。
*   **关键技术细节**：
    *   **两服务器模型**：基于秘密共享（Secret Sharing），将数据库和用户查询分布在两个非共谋的半诚实服务器上。
    *   **加密距离计算**：使用 Beaver Triples 技术在加密状态下计算查询向量与文档向量的点积（余弦相似度）。
    *   **交互式二分协议**：
        1. 用户设定初始距离区间。
        2. 服务器利用**分布式比较函数（DCF）**统计处于当前阈值内的文档数量，并将加密计数返回给用户。
        3. 用户解密计数并根据其与目标 $k$ 的关系更新区间，直至找到合适的阈值 $d_k$。
    *   **恶意用户防御**：引入验证机制，限制二分迭代次数（防止信息泄露累积）和最终返回的结果集大小，确保用户无法通过构造特殊查询来获取超出权限的数据。

### 3. 实验设计
*   **数据集**：使用 BEIR 基准测试中的 `trec-covid` 数据集（包含 171,332 个文档）。
*   **Embedding 模型**：采用 `Cohere/beir-embed-english-v3` 生成 1024 维向量。
*   **Benchmark（对比方法）**：最先进的隐私保护检索系统 **PRAG** (Zyskind et al., 2024)。
*   **评估指标**：检索准确率（Recall）、结果相关性得分（Relevance Score）、服务器处理时间（Latency）、通信开销（Communication Volume）。

### 4. 资源与算力
*   **硬件环境**：实验在一台物理服务器上运行，配备 **2 颗 AMD EPYC 7352 CPU**（共 96 个逻辑核心），**188 GiB 内存**。

### 5. 实验数量与充分性
*   **实验规模**：
    *   测试了不同文档规模 $N$（从 $2^{14}$ 到 $2^{20}$）。
    *   测试了不同检索数量 $k$（从 16 到 1024）。
    *   进行了数值误差分析，验证了定点数转换对 Recall 的影响。
*   **充分性评价**：实验设计较为充分，不仅对比了计算效率，还深入分析了通信轮数（RTT）和安全性边界（信息泄露量）。对比 PRAG 的实验涵盖了从常规 $k$ 到超大 $k$ 的场景，客观展示了二分法在复杂度上的优势。

### 6. 主要结论与发现
*   **性能飞跃**：在 $k=16$ 时，$p^2$RAG 比 PRAG 快 3 倍；当 $k$ 增加到 1024 时，加速比达到 **300 倍**。
*   **扩展性强**：与传统方法性能随 $k$ 增加而剧烈下降不同，$p^2$RAG 的处理时间对 $k$ 的增加不敏感，甚至在大 $k$ 下表现更优。
*   **高准确性**：尽管使用了定点数近似计算，但在测试数据集上的 Recall 依然接近 1.0，且检索结果的相关性得分符合预期。
*   **安全可控**：通过限制迭代次数，将物理信息泄露严格限制在 $O(\log^2 N)$ 比特内。

### 7. 优点
*   **算法范式创新**：巧妙地将 Top-$k$ 排序问题转化为阈值统计问题，规避了安全排序中复杂的比较网络。
*   **支持动态 $k$ 值**：用户可以在查询时灵活决定检索数量，无需重新预处理数据库。
*   **高效并行**：DCF 评估过程天然支持大规模多线程并行，充分利用了现代多核 CPU 的算力。

### 8. 不足与局限
*   **通信轮数（RTT）较多**：二分法是交互式的，需要用户与服务器之间进行多轮通信（约 $\log N$ 轮），在网络延迟较高的环境下（如跨国 WAN）可能会增加总响应时间。
*   **服务器间带宽压力**：在处理百万级数据时，服务器间的通信量（数百 MB）对数据中心内部带宽有一定要求。
*   **功能单一性**：目前仅支持基于向量相似度的检索，尚未涉及关键词过滤等混合检索场景。

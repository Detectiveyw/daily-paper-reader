## 摘要
对云托管大语言模型（LLMs）日益增长的依赖，使得客户端敏感数据（如提示词和响应）面临服务提供商潜在的隐私泄露风险。现有方法难以同时兼顾隐私保护、模型性能和计算效率。为应对这一挑战，我们提出了 Talaria，这是一个机密推理框架，通过对 LLM 流水线进行划分来保护客户端数据，且不损害云端的模型知识产权或推理质量。Talaria 在客户端控制的机密虚拟机（CVM）中执行敏感且与权重无关的操作，同时将与权重相关的计算卸载到云端 GPU。这些环境之间的交互由我们提出的可逆掩码外包（ReMO）协议保障，该协议采用混合掩码技术在计算外包前对中间数据进行可逆模糊处理。广泛的评估表明，Talaria 能够防御最先进的令牌推理攻击，将令牌重构准确率从 97.5% 以上降低至平均 1.34%；同时，作为一种无损机制，它能保证输出与原始模型完全一致，且不会显著降低效率和可扩展性。据我们所知，这是首项在确保云端无法访问客户端提示词和响应的同时，兼顾模型隐私、性能和效率的工作。

## Abstract
The increasing reliance on cloud-hosted Large Language Models (LLMs) exposes sensitive client data, such as prompts and responses, to potential privacy breaches by service providers. Existing approaches fail to ensure privacy, maintain model performance, and preserve computational efficiency simultaneously. To address this challenge, we propose Talaria, a confidential inference framework that partitions the LLM pipeline to protect client data without compromising the cloud's model intellectual property or inference quality. Talaria executes sensitive, weight-independent operations within a client-controlled Confidential Virtual Machine (CVM) while offloading weight-dependent computations to the cloud GPUs. The interaction between these environments is secured by our Reversible Masked Outsourcing (ReMO) protocol, which uses a hybrid masking technique to reversibly obscure intermediate data before outsourcing computations. Extensive evaluations show that Talaria can defend against state-of-the-art token inference attacks, reducing token reconstruction accuracy from over 97.5% to an average of 1.34%, all while being a lossless mechanism that guarantees output identical to the original model without significantly decreasing efficiency and scalability. To the best of our knowledge, this is the first work that ensures clients' prompts and responses remain inaccessible to the cloud, while also preserving model privacy, performance, and efficiency.

### 1. 核心问题与研究动机
*   **核心问题**：在云端大语言模型（LLM）推理过程中，客户端的敏感数据（提示词 Prompt 和模型响应 Response）完全暴露给云端服务商，存在严重的隐私泄露风险。
*   **研究动机**：现有的隐私保护方案面临“不可能三角”挑战：
    1.  **模型隐私**：保护客户端数据时不应泄露云端模型的知识产权（权重）。
    2.  **模型性能**：隐私机制不应改变模型输出（需满足输出不变性）。
    3.  **模型效率**：隐私机制引入的计算开销必须在可接受范围内。
*   **背景**：现有的差分隐私（DP）会降低输出质量，同态加密（HE）开销巨大，而现有的机密虚拟机（CVM）方案要么无法扩展，要么会泄露模型权重或响应内容。

### 2. 方法论：Talaria 框架
*   **核心思想**：提出**机密分区推理（Confidential Partitioned Inference, CPI）**。将 LLM 推理流水线拆分为两部分：
    *   **权重相关操作（Weighted Decoding）**：如线性投影，在拥有权重的云端 GPU 上执行。
    *   **结构化/敏感操作（Structural Decoding）**：如归一化、注意力分数计算、残差连接和采样，在客户端控制的机密虚拟机（CVM）中执行。
*   **关键技术：可逆掩码外包协议（ReMO）**：
    *   **混合掩码（Hybrid Masking）**：为了在不泄露权重 $W$ 的前提下让 CVM 能够还原结果，协议将掩码 $M$ 分为公共基 $M_{pub}$ 和私有混合矩阵 $M_{pvt}$。
    *   **流程**：
        1.  **初始化**：CVM 生成 $M_{pub}$ 发给 GPU，GPU 返回预计算的 $R_{pub} = M_{pub}W$。
        2.  **推理时**：CVM 对中间嵌入 $E$ 加掩码 $\hat{E} = E + M$（其中 $M = M_{pvt}M_{pub}$），发给 GPU。
        3.  **计算与还原**：GPU 计算 $\hat{O} = \hat{E}W$ 并返回。CVM 利用本地秘密通过 $O = \hat{O} - M_{pvt}R_{pub}$ 精确还原出原始计算结果 $EW$。
*   **安全性保障**：通过 CVM 的远程度量（Attestation）确保代码完整性，防止客户端逆向模型权重，同时通过掩码确保云端无法获取明文数据。

### 6. 主要结论与发现
*   **极强的隐私保护**：将令牌重构准确率（TRA）从 97.5% 以上降低到平均 **1.34%**，语义相似度（CSS）降至 0.47 以下，使攻击者几乎无法获取有效信息。
*   **完全无损**：Talaria 保证了输出与原始模型位对位（Bit-for-bit）一致，解决了 DP 方案带来的性能下降问题。
*   **高效且可扩展**：相比于在 CVM 中运行完整模型的方案，Talaria 的端到端延迟更低，且随着模型参数增加，其性能优势更加明显。
*   **双向保护**：成功实现了在保护用户隐私的同时，也保护了云端模型的权重不被客户端逆向。

### 7. 优点与亮点
*   **首创性**：据作者称，这是首个同时保护提示词和响应隐私，且兼顾模型隐私、性能和效率的工作。
*   **数学严谨性**：提出了 ReMO 协议，并给出了信息论层面的安全界限证明。
*   **实用性强**：利用了现有的硬件安全特性（CVM/GPU CC），无需对 LLM 架构进行修改，具有较好的部署前景。
*   **无损输出**：通过代数还原实现了 100% 的推理精度保持。

### 8. 不足与局限
*   **威胁模型限制**：主要针对“诚实但好奇”（Honest-but-curious）的云端服务商，未深入探讨完全恶意（Fully Malicious）且可能篡改协议流程的攻击者。
*   **部分模型保护**：虽然保护了权重，但模型结构（算子顺序等）对客户端是可见的（尽管作者认为这不属于核心 IP 泄露）。
*   **硬件依赖**：高度依赖支持机密计算的 GPU 和 CVM 环境，在老旧硬件上无法运行。
*   **通信开销**：虽然计算效率高，但 CVM 与 GPU 之间频繁的中间数据交换可能在网络带宽受限的环境下产生瓶颈。

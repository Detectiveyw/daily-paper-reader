## 摘要
隐私保护机器学习即服务 (PP-MLaaS) 通过集成同态加密 (HE) 和多方计算 (MPC) 等密码学原语，实现了安全的神经网络推理，从而保护了客户端数据和服务器模型。最近的混合原语框架显著提高了推理效率，但它们按顺序处理批处理输入，在优先处理紧急请求方面缺乏灵活性。朴素的插队方式会引入可观的计算和通信开销，增加了队列中已有输入的不可忽视的延迟。我们开启了对批处理推理中隐私保护插队的研究，并提出了 PrivQJ，这是一个能够在不降低系统整体性能的情况下实现高效优先级处理的新型框架。PrivQJ 通过处理中的槽位回收利用输入间的共享计算，允许优先输入以几乎零额外密码学成本的方式搭载到正在进行的批处理计算中。理论分析和实验结果均表明，与最先进的 PP-MLaaS 系统相比，该方案将开销降低了一个数量级以上。

## Abstract
Privacy-Preserving Machine Learning as a Service (PP-MLaaS) enables secure neural network inference by integrating cryptographic primitives such as homomorphic encryption (HE) and multi-party computation (MPC), protecting both client data and server models. Recent mixed-primitive frameworks have significantly improved inference efficiency, yet they process batched inputs sequentially, offering little flexibility for prioritizing urgent requests. Naïve queue jumping introduces considerable computational and communication overhead, increasing non-negligible latency for in-queue inputs.   We initiate the study of privacy-preserving queue jumping in batched inference and propose PrivQJ, a novel framework that enables efficient priority handling without degrading overall system performance. PrivQJ exploits shared computation across inputs via in-processing slot recycling, allowing prior inputs to be piggybacked onto ongoing batch computation with almost no additional cryptographic cost. Both theoretical analysis and experimental results demonstrate over an order-of-magnitude reduction in overhead compared to state-of-the-art PP-MLaaS systems.

### 1. 核心问题与整体含义（研究动机和背景）
*   **背景**：PP-MLaaS 结合了同态加密（HE）和多方计算（MPC）来保护客户端数据和服务器模型。为了提高效率，现有框架通常采用“批处理”模式。
*   **核心问题**：现有的批处理推理系统是顺序执行的，缺乏处理紧急请求（高优先级输入）的灵活性。
*   **研究动机**：传统的“插队”方式（Hard Queue Jumping）要么需要重新打包批次，要么需要丢弃当前队列中的任务，这会产生巨大的计算和通信开销，导致队列中原有任务的延迟增加（有时甚至增加数小时或数天）。本文旨在实现一种“近乎零成本”的插队机制，在不损害系统整体性能的前提下优先处理紧急任务。

### 2. 方法论：核心思想与关键技术
*   **核心思想**：**处理中槽位回收（In-processing slot recycling）**。利用 HE 密文在 SIMD（单指令多数据）操作中未被填满的闲置槽位（Slots），将优先输入的计算任务“搭载”在正在进行的批处理计算中。
*   **关键技术细节**：
    *   **密文槽位利用**：HE 密文（如 BFV 方案）通常包含数千个槽位（如 $N=8192$）。在处理卷积等操作时，由于数据对齐要求，往往存在大量闲置槽位。
    *   **链式批处理（Chained Batching）**：PrivQJ 将优先输入的计算视为“拼图”，将其拆分并填充到当前批次多个输入的闲置槽位中。
    *   **在线/离线回收策略**：
        *   **在线阶段**：在 ReLU-卷积（fcfr）组合层中，利用中间密文 $t$ 的尾部槽位传递优先输入的数据。
        *   **离线阶段**：在矩阵乘法转换（im2col）后的密文 $cr_0$ 中回收槽位，预先计算优先输入的线性部分。
    *   **流程优化**：改进了混合原语推理流程，将两次卷积操作合并为一次，减少了在线通信轮数（仅需 0.5 轮额外通信即可完成插队任务的份额交换）。

### 6. 主要结论与发现

*   **效率均衡**：在提供插队灵活性的同时，没有牺牲普通批处理任务的吞吐量。

### 8. 不足与局限
*   **模型覆盖面**：目前主要针对卷积神经网络（CNN），虽然文中提到可以扩展，但尚未在 Transformer 或大语言模型（LLM）上进行验证。
